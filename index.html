Comandos

Slash Commands — O que você pode digitar

Digite esses comandos diretamente no chat do Claude Code para acionar funcionalidades especiais.

/help

Ajuda geral

Exibe todos os comandos disponíveis, atalhos de teclado e informações sobre o Claude Code.

/clear

Limpar conversa

Apaga o histórico da conversa atual. Útil quando quer começar do zero em um novo contexto.

/config

Configurações

Abre o painel de configurações: tema, modelo (Opus/Sonnet/Haiku), preferências visuais.

/fast

Modo rápido

Ativa o Fast Mode com Claude Opus para respostas mais rápidas. Disponível no Opus 4.6 e 4.7.

/review

Revisar Pull Request

Revisa o PR atual ou um número específico: /review 42. Analisa bugs, segurança e qualidade.

/ultrareview

Revisão multi-agente

Revisão aprofundada usando vários agentes em paralelo. Use /ultrareview <PR#> para um PR do GitHub.

/init

Inicializar CLAUDE.md

Cria ou atualiza o arquivo CLAUDE.md com documentação do projeto. Essencial para contexto persistente.

/schedule

Agendar tarefas

Cria agentes que rodam em horários definidos (cron). Ex: verificar deploys toda manhã.

/loop

Loop repetitivo

Executa um comando repetidamente. Ex: /loop 5m /review revisa a cada 5 minutos.

/run

Rodar o projeto

Inicia o servidor/app do projeto atual e observa o comportamento real da aplicação.

/verify

Verificar mudança

Roda o app e verifica se uma mudança de código funciona corretamente na prática.

/security-review

Revisão de segurança

Analisa o código do branch atual em busca de vulnerabilidades (XSS, SQLi, OWASP Top 10).

Skills

Skills Disponíveis — Capacidades Especializadas

Skills são agentes especializados que o Claude ativa automaticamente ou quando você usa um comando. Cada um tem conhecimento profundo em uma área.

⚙️

update-config

Configura o Claude Code via settings.json. Use para permissões, variáveis de ambiente, hooks automáticos ("sempre que X acontecer, faça Y").

Quando usar: "permitir comandos npm", "quando Claude parar, mostre notificação"
⌨️

keybindings-help

Personaliza atalhos de teclado. Modifica ~/.claude/keybindings.json para criar chord bindings, rebinds e atalhos customizados.

Quando usar: "mudar Ctrl+S", "adicionar atalho para /review"
✅

verify

Roda o app real e observa o comportamento. Testa o caminho feliz e casos especiais. Não apenas compila — verifica visualmente na prática.

Quando usar: "verifique se o login funciona", "confirme se o fix resolveu"
🔍

code-review

Revisa o diff atual buscando bugs de correção. Níveis: low/medium (alta confiança) ou high/max (cobertura ampla). Pode postar como comentários inline no PR.

Quando usar: antes de mergear, /review --comment para postar no GitHub
🔒

fewer-permission-prompts

Escaneia seus logs e cria uma allowlist no settings.json para reduzir as confirmações repetitivas de permissão.

Quando usar: quando Claude pede permissão para as mesmas coisas toda vez
🔄

loop

Executa um comando ou skill repetidamente em intervalos. Ex: /loop 5m /security-review para monitoramento contínuo.

Quando usar: monitorar CI, verificar deploy, polling de status
📅

schedule

Cria agentes remotos agendados (cron). Pode configurar execuções únicas ou recorrentes. Ex: "rodar às 3pm", "verificar toda manhã".

Quando usar: automações periódicas, relatórios automáticos
🤖

claude-api

Constrói, depura e otimiza apps usando a Claude API / Anthropic SDK. Inclui prompt caching, migração entre versões de modelo, tool use e mais.

Ativado quando: código importa anthropic ou @anthropic-ai/sdk
▶️

run

Inicia e controla o app do projeto para ver mudanças funcionando. Cobre CLI, servidor, Electron, browser-driven e outros tipos de projeto.

Quando usar: "rode o projeto", "mostre como ficou", "screenshot do app"
📝

init

Inicializa o arquivo CLAUDE.md com documentação da base de código. Essencial para dar contexto persistente ao Claude sobre seu projeto.

Quando usar: início de projeto novo ou ao integrar Claude em projeto existente
👁️

review

Revisa um Pull Request completo — não só o último commit, mas todos os commits do branch. Gera título, resumo e plano de testes.

Quando usar: antes de submeter um PR, para preparar a descrição
🛡️

security-review

Revisão completa de segurança das mudanças pendentes. Verifica OWASP Top 10: XSS, SQLi, injeção de comando, autenticação, e mais.

Quando usar: antes de deploy em produção, ao mexer em auth ou input de usuário
Integrações

Outros Apps que Potencializam o Claude Code

Conecte o Claude Code a outras ferramentas para multiplicar sua produtividade.

VS Code

Extensão oficial

Integração nativa com o editor mais popular. O Claude Code aparece como painel lateral, pode ler arquivos abertos, ver erros do editor e aplicar mudanças diretamente.

Lê arquivos abertos Aplica edições Ver diagnósticos Painel lateral
Instale a extensão "Claude Code" no marketplace do VS Code
O painel Claude aparece na barra lateral esquerda
Claude lê automaticamente o arquivo que você está editando
Peça mudanças em linguagem natural — Claude aplica diretamente

JetBrains IDEs

IntelliJ, PyCharm, WebStorm...

Plugin para toda a família JetBrains. Funciona em IntelliJ IDEA, PyCharm, WebStorm, GoLand, Rider e mais. Mesmo nível de integração do VS Code.

IntelliJ IDEA PyCharm WebStorm GoLand
Abra File → Settings → Plugins
Busque por "Claude Code" e instale
Reinicie o IDE e o painel aparece na lateral direita

GitHub

PRs, Issues, Actions

Claude Code usa o CLI gh para criar PRs, comentar em issues, verificar status de CI, criar releases e gerenciar o repositório.

Criar PRs Comentar Issues CI/CD Status gh CLI
# Claude faz isso automaticamente quando você pede:
$ gh pr create --title "feat: login"
$ gh issue comment 42 --body "..."
✓ PR criado: github.com/user/repo/pull/15

MCP Servers

Model Context Protocol

MCP (Model Context Protocol) permite que Claude acesse ferramentas externas. Há servidores prontos para dezenas de serviços.

Slack Linear Notion Figma Jira Postgres
MCP servers rodam localmente como processos separados
Configure no settings.json via /update-config
Claude ganha acesso a ler/escrever dados nessas plataformas

Dicas Pro

Como Trabalhar Melhor com o Claude Code

Técnicas para extrair o máximo e evitar os erros mais comuns.

📋

Use o CLAUDE.md — O "manual" do seu projeto

O arquivo CLAUDE.md na raiz do projeto é carregado automaticamente em toda conversa. Coloque lá as regras do projeto, stack, convenções e informações importantes.

Dica: rode /init para criar um CLAUDE.md automaticamente baseado no seu código atual.
# Exemplo de CLAUDE.md
## Stack: Next.js 14, TypeScript, Tailwind, Supabase
## Sempre use componentes funcionais
## Nunca commitar .env
## Testes com Jest + Testing Library
🎯

Seja específico — contexto é tudo

Quanto mais contexto você der, melhor o resultado. Inclua: o objetivo, as restrições, o que já tentou e o que não funciona.

Evite: "corrija o bug"
Prefira: "o botão de login na página /login está retornando 401 quando o email tem letra maiúscula — o backend espera lowercase"
⚡

Tarefas independentes em paralelo

O Claude pode disparar múltiplos agentes ao mesmo tempo para tarefas que não dependem uma da outra.

você: crie os testes unitários E a documentação
# Claude roda os dois em paralelo → mais rápido
Dica: use "E" ou "ao mesmo tempo" para sinalizar paralelismo.
🔑

Reduza popups de permissão

Se o Claude fica pedindo permissão para as mesmas operações repetidamente, use a skill para criar uma allowlist.

você: use /fewer-permission-prompts para configurar
✓ Allowlist criada em .claude/settings.json
Alternativa: diga "permita comandos npm" para adicionar permissão específica.
🧠

Claude tem memória persistente

Claude lembra informações entre conversas diferentes. Você pode pedir para salvar preferências, contexto de projeto e feedbacks.

você: lembre que prefiro código sem comentários
você: lembre que este projeto usa Tailwind v4
✓ Salvo na memória do projeto
🌿
